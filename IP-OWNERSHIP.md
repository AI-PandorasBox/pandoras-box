## Intended structure

Once finalised, the IP framework is intended to take the following form:

| Asset | Intended owner |
|---|---|
| Source code in this repository | Hope Inside Limited (UK company number 17228483) — licensed publicly under Apache 2.0 |
| "Pandora's Box" word mark and logo marks | Same holding entity — see [TRADEMARK.md](TRADEMARK.md) |
| Domain `ai-pandorasbox.co.uk` | Same holding entity |
| Published documentation, hero imagery, screenshots, marketing copy | Same holding entity, licensed under Creative Commons (specific variant TBD) |
| Outputs generated by the software when run by an operator | The operator — Pandora's Box maintainers claim no ownership over what users generate with the software |

## Why a single holding entity

Several operating businesses have contributed in different ways during the build:

- **Hardware hosting** — one operating business has provided the development hardware (a single Mac mini) on which the system was developed.
- **API tokens for tenant-specific features** — operating businesses have paid for API tokens consumed during the development of features specific to those tenants (CRM integrations, marketing automations, web design wizards, etc.).
- **General development funding** — the framework and core primitives have been funded personally by the project maintainer.

To prevent any future ambiguity about ownership of the framework code, the framework is treated as wholly owned by Hope Inside Limited. Operating businesses that funded specific tenant features receive non-exclusive, non-transferable, perpetual licences from Hope Inside Limited to use those features internally. Operating businesses that hosted hardware are positioned as service vendors to Hope Inside Limited rather than co-owners of the software.

These arrangements will be formalised in inter-company licence agreements, subject to solicitor review.

## Contributor License Agreement

Pull requests to this repository require a signed Contributor License Agreement before they can be merged. See [CONTRIBUTING.md](CONTRIBUTING.md) for the process. The CLA grants the project a perpetual, worldwide, non-exclusive licence to use your contribution under Apache 2.0 — you retain copyright in your work; the project gains the right to distribute it.

## What this means for users and forkers

- **Users running the software:** no special action required. The Apache 2.0 licence applies as-is. You own what you produce; Pandora's Box maintainers accept no liability for what you do with the software (see [DISCLAIMER.md](DISCLAIMER.md)).
- **Operators integrating with paid third-party services** (Anthropic, Microsoft, Google, ElevenLabs, IG.com, others): you are bound by those third parties' terms; Pandora's Box accepts no liability for your compliance.
- **Forkers redistributing modified versions:** you may distribute under Apache 2.0 freely, but you must NOT use the "Pandora's Box" name or logos for your fork (see [TRADEMARK.md](TRADEMARK.md)). Pick a different product name.
- **Commercial users:** you may use the software commercially under Apache 2.0. Use of the trademark for commercial activity requires written permission.

## What this does NOT do

This document does not transfer or assign any rights between any parties. It is a statement of intent only. Until the supporting inter-company licences and assignments are executed by the relevant parties, no rights change hands from what they were before this document existed.
