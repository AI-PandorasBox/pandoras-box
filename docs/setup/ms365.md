# Setup — Microsoft 365 (Outlook / Calendar / SharePoint)

<!-- _A2_INSTALLER_AND_GUIDES_V1 -->

For each business tenant in Pandora's Box that runs Mail / Calendar / Files modules against Microsoft 365, you register the project once as an "app" in your Azure Active Directory tenant. This is a one-time, ~10-minute setup per business.

## Prerequisites

- A Microsoft 365 subscription with admin access to register apps (Global Administrator or Application Administrator role).
- An Azure portal account — comes free with any M365 subscription; sign in at https://portal.azure.com.

## What you'll end up with

- A registered Azure AD application with: client ID, tenant ID, client secret.
- These three values stored in macOS Keychain (the installer prompts for them).
- Token cache files under `/opt/pandoras-box/<tenant>/ms365-auth/` populated automatically on first agent run.

## Setup steps

### 1. Open Azure portal

https://portal.azure.com — sign in with the M365 admin account.

### 2. Navigate to App registrations

Search box at the top → type "App registrations" → click the matching service.

### 3. Create a new registration

- **Name**: `Pandora's Box — <Company Name>` (visible only to your admins)
- **Supported account types**: "Accounts in this organizational directory only" (single tenant)
- **Redirect URI**: leave blank for now (the installer doesn't need a callback for the device-flow auth Pandora's Box uses)
- Click **Register**

### 4. Capture the IDs

On the overview page that loads after registration, copy:
- **Application (client) ID** — looks like `12345678-aaaa-bbbb-cccc-ddddeeeeffff`
- **Directory (tenant) ID** — same shape

You'll paste these into the installer prompt later.

### 5. Add API permissions

In the left nav: **API permissions** → **Add a permission** → **Microsoft Graph** → **Delegated permissions**.

Add the following scopes:

| Scope | What it allows |
|---|---|
| `Mail.ReadWrite` | Read and draft emails for the tenant |
| `Mail.Send` | Send emails (only after operator approval in Pandora's Box) |
| `Calendars.ReadWrite` | Read and create calendar events |
| `Files.ReadWrite.All` | Read and write SharePoint and OneDrive files |
| `User.Read` | Read the signed-in user's basic profile (required) |
| `offline_access` | Refresh tokens without re-auth (required) |

Click **Add permissions**. Then click **Grant admin consent for <your tenant>** — required for organisational scopes.

### 6. Generate a client secret

In the left nav: **Certificates & secrets** → **Client secrets** → **New client secret**.

- Description: `Pandora's Box`
- Expires: 24 months (or per your security policy)
- Click **Add**

**Copy the secret value immediately** — Azure shows it only once. If you miss it, generate a new one.

### 7. Run the Pandora's Box installer

When the installer reaches the Microsoft 365 step for the relevant tenant, paste the three values when prompted:

```
Tenant ID: 12345678-aaaa-bbbb-cccc-ddddeeeeffff
Client ID: 87654321-aaaa-bbbb-cccc-ddddeeeeffff
Client Secret: ********************
```

The installer stores all three in macOS Keychain entries scoped to the tenant. Token caches are populated on first agent run via device-flow authentication.

## Verifying it works

After the installer finishes, send a test message to your business agent:

```
<your-tenant-slug>: list today's emails
```

(Substitute your tenant slug.) You should see a structured list. If you get a 401, the installer's keychain values are wrong; re-run the MS365 step.

## Multi-mailbox scenarios (one tenant, multiple addresses)

If your business tenant runs multiple mailboxes (e.g. shared `Office@example.com` plus a personal `name@example.com`), Pandora's Box can authenticate against each separately. The installer asks "Add another mailbox to this tenant?" after the first mailbox completes — pick yes and supply the additional mailbox's auth.

## Revoking access

In Azure portal: App registrations → your app → **Delete**. This invalidates all tokens issued under that registration immediately. Mail / Calendar / Files modules in the corresponding Pandora's Box tenant stop working until you create a fresh registration and re-run the installer's MS365 step.
